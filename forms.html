It is possible to use the :valid and :invalid CSS pseudo-classes to style a form element based on whether the elements inside the form are valid. The :valid CSS pseudo-class represents an element that has been validated and is considered to be valid. The :invalid CSS pseudo-class represents an element that has been validated and is considered to be invalid.

Input "Restrictions"

Here is a list of some common input restrictions attributes:

(1)"checked"
Specifies that an input field should be pre-selected when the page loads (for type="checkbox" or type="radio")

(2)"maxlength"
Specifies the maximum number of characters allowed in an input field

(3)"min"
Specifies the minimum value for an input field

(4)"max"
Specifies the maximum value for an input field

(5)"pattern"
Specifies a regular expression that an input field's value is checked against

(6)"required"
Specifies that an input field must be filled out before submitting the form

(7)"size"
Specifies the width, in characters, of an input field

(8)"step"
Specifies the legal number intervals for an input field

(9)"placeholder"
Specifies a short hint that describes the expected value of an input field

(10)"readonly"
Specifies that an input field is read-only (cannot be changed)

(11)"disabled"
Specifies that an input field should be disabled

(12)"autofocus"
Specifies that an input field should automatically get focus when the page loads

(13)"autocomplete"
Specifies whether an input field should have autocomplete on or off. Example for min, max, step below.
